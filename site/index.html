结果承诺 Result Promise

用户先看到结论，再看到图，再决定要不要导出产物给团队。

这就是结果页的默认阅读顺序: base-case result、cost-effectiveness 判断、uncertainty、calibration fit，以及可交付产物。

Default readout

一次 run 结束后，先回答“值不值得”“拟合得够不够好”“能不能直接交付”。

结果页不会只给你一堆图。它会先把 ICER、QALY、threshold judgment 和 calibration status 放在最前面， 然后再进入 PSA scatter、CEAC、cohort trace 和 artifacts。

查看完整 Example Run 打开 Results 页面
Calibration overlay

先确认模型是否贴近 observed data

默认展示 observed vs predicted overlay，并给出 RMSE 或 goodness-of-fit，帮助你判断校准是否可接受。

Uncertainty

用 PSA scatter 和 CEAC 解释不确定性

不是只给一个 ICER，而是直接告诉你样本分布和在阈值下的 cost-effective 概率。

Cohort trace

看患者队列什么时候开始明显转移

动态 Markov 与 cohort trace 会一起出现，帮助你判断状态流动是否符合临床逻辑。

Export pack

把结果变成可以交付的 review bundle

支持导出 Excel summary、calibration pack、review bundle 和 run metadata，方便团队复核与汇报。
